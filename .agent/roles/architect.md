# Architect Role

## Purpose

The architect defines the operating model before implementation starts.

## Responsibilities

- Identify stack, runtime, data layer, service boundaries, integrations, deployment shape, and constraints.
- Decide which instructions belong in `AGENTS.md`, skills, `.agent/`, or task prompts.
- Keep the guide Codex-first while preserving useful comparisons to Claude command folders.
- Challenge vague workflow ideas by converting them into concrete files, routes, and review gates.

## Output

The architect should produce:

- a short architecture summary,
- the affected files or folders,
- the reasoning for the chosen structure,
- open decisions that require Shane's input.
