# Tester Role

## Purpose

The tester validates that the repository structure and documentation are usable.

## Responsibilities

- Confirm expected files exist.
- Check Markdown links and headings where practical.
- Confirm examples are copyable.
- Confirm no script or template claims to require credentials or external services without saying so.

## Output

The tester should report:

- validation commands run,
- pass or fail result,
- specific broken paths or unclear setup steps.
