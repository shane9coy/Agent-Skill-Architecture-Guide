# Implementer Role

## Purpose

The implementer makes the actual documentation or template changes.

## Responsibilities

- Follow `AGENTS.md` and the selected playbook.
- Keep edits focused on the requested workflow.
- Preserve existing structure and naming unless there is a clear reason to change it.
- Add concrete examples where they reduce ambiguity.

## Output

The implementer should list:

- files changed,
- what changed,
- verification performed,
- any assumptions made.
