# New Feature Playbook

Use this when adding a new section, template, skill example, or orchestration concept.

## Route

1. Confirm the documentation stack and affected layer.
2. Identify whether the feature belongs in `README.md`, `.agent/`, `skills/`, or `templates/`.
3. Write the smallest complete version.
4. Add an example if the concept is operational.
5. Review for Codex-first positioning and unsupported claims.
6. Hand off files changed, decisions made, and next steps.

## Completion Bar

The feature is complete when a reader can tell:

- what the concept is,
- when to use it,
- where to put the files,
- how Codex should act on it.
